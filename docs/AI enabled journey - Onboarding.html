AI-Enabled Transformation Journey

Accelerating enterprise change using SMBC Corporate Onboarding as the pilot case.

AI-Powered Capabilities: Showcasing the full end-to-end transformation of an enterprise journey.
What will be done

We are demonstrating a full 6-step transformation pipeline driven by AI, reducing an historically painful 90-180 day client onboarding process down to ~5 days.

  • Step 1 (As-Is): Map current inefficiencies automatically.
  • Step 2 (To-Be): Design an optimal, agent-driven workflow.
  • Step 3 (Solutioning): Architect a solution respecting legacy constraints.
  • Step 4 (Prototype): Generate production-ready frontend UI.
  • Step 5 (Schema): Establish the relational data orchestration layer.
  • Step 6 (WBS): Output the execution plan and work breakdown.

1. Map the As-Is Process

Automated discovery of current state pain points.

AI Can... synthesize massive qualitative inputs into a structural understanding of current processes.
What will be done

Instead of lengthy manual discovery workshops, we feed multimodal data into the AI to build the current state map in minutes.

  • Industry Research: NotebookLM instantly compiles industry benchmarks, regulatory requirements, and existing best practices from public and proprietary documents in just a few minutes.
  • Process Owner Interviews: AI ingests transcripts from process owners to map out the actual (shadow) processes, bottlenecks, and manual workarounds occurring today.
  • Output: A highly accurate mapping of the current 16-29 week process, identifying exactly where friction occurs.

2. Design the To-Be Process

Redrafting the journey with AI-agent accelerators.

AI Can... redesign sequential human workflows into parallel, agent-orchestrated operations.
What will be done

The AI redrafts the journey map by inserting targeted agents to handle administrative friction, allowing human operators to focus entirely on judgment calls.

  • UBO Intelligence Agent: Recursively queries registries to automatically unwrap BVI shell companies and find natural persons.
  • Document Extraction Agent: OCR reads uploaded PDFs and cross-checks against registry data in real-time.
  • Credit Drafter Agent: Spreads financials automatically and authors the credit memo for RM review.

3. AI Solutioning

Designing an architecture that respects enterprise boundaries.

AI Can... propose a technical architecture that bridges modern AI with legacy banking constraints.
What will be done

The AI solutions a stack capable of delivering the To-Be process without requiring a core banking replacement.

  • Front-End Portal: Next.js and Tailwind to provide dual-persona UX for the Client Portal and RM Cockpit.
  • Orchestration Layer: Supabase acts as the agile middle layer, managing shared application states and tool-calling for AI prompts.
  • AI Engine: Claude API integrated securely for Copilot and Credit Memo drafting.
  • Legacy Cores: The orchestration layer pushes/pulls data (Read/Write) to existing systems (e.g. Fenergo) to maintain compliance.

5. Data Structures & Schema

Defining the relational backbone of the onboarding journey.

AI Can... architect a robust database schema to support complex enterprise workflows.
What will be done

The AI generates a Star-schema inspired Postgres structure optimized for the orchestration layer, connecting the central 'Application' fact table to all related dimensions (Entities, UBOs, Products).

  • Fact: The Onboarding Application state.
  • Dimensions: Organizations, Entities (supporting self-referencing hierarchy for trees), Natural Persons (UBOs), and Documents.
CREATE TABLE applications ( -- FACT TABLE
  id uuid PK,
  org_id uuid FK,
  status text, stage int
);

CREATE TABLE entities ( -- DIMENSION
  id uuid PK,
  app_id uuid FK,
  parent_id uuid FK -- hierarchy,
  legal_name text,
  is_shell boolean
);

CREATE TABLE ubos ( -- DIMENSION
  id uuid PK,
  app_id uuid FK,
  ownership_pct numeric,
  screening_status text
);

6. WBS Generation

AI-assisted project planning and execution tracking.

AI Can... translate the technical architecture and UI requirements into a structured delivery sprint.
What will be done

Based on the delta between the As-Is and To-Be states, the AI formulates a comprehensive Work Breakdown Structure.

  • Phase 0 (Foundation): Initializing Next.js, configuring Supabase, and mapping schemas to support the core banking logic.
  • Phase 1 (Cockpit): Building out the RM dashboard and populating it with realistic background seed data.
  • Phases 2-4 (AI Implementation): Connecting the Claude API, configuring the Copilot sidecar, and integrating the Document Extraction hooks.
  • Phase 5 (Polish): Final end-to-end testing, refining the perpetual KYC UI, and preparing the hand-over.
