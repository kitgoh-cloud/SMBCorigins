<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `npm registry → local node_modules` | npm-published packages cross into the repo's build pipeline. `create-next-app` itself, plus all transitive deps. |
| `Vercel Dashboard → Vercel build pipeline` | Env vars set in dashboard cross into Next.js build context. `NEXT_PUBLIC_USE_MOCK` is intentionally exposed to the browser bundle. |
| `Local CLAUDE.md → create-next-app` | create-next-app writes files into the working directory; an unguarded run could overwrite the canonical Phase 1 closure CLAUDE.md. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | npm dependency tree | mitigate | All versions pinned by `create-next-app@16.2.4` itself; `package-lock.json` commits the exact resolution; CI in 02-05 uses `npm ci` which fails on lockfile drift. ASVS L1 dependency-integrity check satisfied by lockfile + pinned major. |
| T-02-02 | Tampering | Existing CLAUDE.md | mitigate | `--no-agents-md` flag in Task 3 + pre-flight backup to `/tmp/CLAUDE.md.phase1-backup` + post-flight `diff` verification. If diff is non-zero, task auto-restores from backup and fails loudly. |
| T-02-03 | Information Disclosure | `NEXT_PUBLIC_USE_MOCK` env var | accept | `NEXT_PUBLIC_*` is intentionally public (Next.js convention — value exposed in browser bundle). Value is `true` (a boolean, no PII). Documented in 02-CONTEXT.md D-35 and CLAUDE.md. **Critical convention for downstream phases:** never put secrets behind `NEXT_PUBLIC_*` (Supabase URL is OK as public; Supabase service-role keys are NOT — must remain server-only without that prefix in Phase 3+). |
| T-02-04 | Elevation of Privilege | Vercel GitHub App permissions | accept | Vercel GitHub App requires read access to the repo to deploy — this is the standard integration model documented at vercel.com/docs/git/vercel-for-github. No write access requested. Stakeholder demo prototype. |
| T-02-05 | Denial of Service | Build pipeline (Turbopack OOM) | accept | Turbopack is the create-next-app default in 16.x; Vercel's free tier has documented build limits. Demo prototype scope; if hit, fall back to `next build` without `--turbopack`. |

All threats LOW severity for a stakeholder demo prototype with no auth, no user data, no DB. No HIGH-severity threats introduced by this plan.
</threat_model>
