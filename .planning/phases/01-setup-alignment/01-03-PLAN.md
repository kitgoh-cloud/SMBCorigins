```
# CODEOWNERS — cross-GSD boundary enforcement
#
# Auto-requests reviewers on PRs touching the four shared boundary files.
# Backed by CONTRACT.md (human prose) and DECISIONS.md (canonical log,
# specifically D-13, D-16, D-24).
#
# Usernames below are PLACEHOLDERS until Kit confirms the real GitHub handles
# in the Phase 1 closure PR (kit/phase-1-alignment). On confirmation, replace
# @kit-placeholder and @evan-placeholder globally.
#
# GitHub matches these patterns against changed paths in the order they
# appear; the LAST matching line wins. Specific paths are listed after
# generic ones for that reason.

# Co-owned: typed boundary surface — both must review every change
types/origin.ts          @kit-placeholder @evan-placeholder
lib/api.ts               @kit-placeholder @evan-placeholder

# Kit-owned: mock implementation — Evan reviews
lib/api.mock.ts          @kit-placeholder @evan-placeholder

# Evan-owned: real implementation — Kit reviews
lib/api.real.ts          @evan-placeholder @kit-placeholder
```

Notes on the pattern syntax:
- Each line is `<path-pattern> <space-separated-owners>`.
- For co-owned files (`types/origin.ts`, `lib/api.ts`), GitHub will request review from EVERY listed handle.
- For single-owned files, the FIRST handle is conventionally the owner; the second is the cross-reviewer. GitHub does not distinguish "owner" vs "reviewer" semantically — both are auto-requested. This matches the contract intent: the file owner writes; the cross-reviewer reviews. Convention is enforced socially, not by the file.
- No `*` (catch-all) line is included — branch protection (D-08) is the gate for everything else. Only the boundary files need cross-GSD auto-routing.

Use the Write tool, not heredoc/cat.
