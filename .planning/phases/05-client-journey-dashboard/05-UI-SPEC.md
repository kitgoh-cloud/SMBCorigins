## Container & Page Layout

The dashboard renders inside `<ClientShell>` (existing Phase 4 component, unchanged):

```
┌─ <body>
│  ├─ <TopStrip /> (56px tall, sticky, trad-green-deep — Phase 4)
│  ├─ <ClientShell>                                ← max-w-[1200px], pt-9 px-10 pb-20
│  │   └─ <ClientJourneyPage>                     ← Server Component, fetches detail
│  │       ├─ <KanjiWatermark>                    ← absolute, top -40 right -60, 360px 縁
│  │       ├─ <GreetingBlock />                   ← free-floating; mb-7 (28px)
│  │       ├─ <ApplicationCard>                   ← .card .card--hero; mb-6 (24px)
│  │       │   ├─ Left column (1.4fr): timeline
│  │       │   └─ Right column (1fr): products + jurisdictions + days-in-progress
│  │       │       (vertical hairline divider 1px mist between columns)
│  │       ├─ <div grid 1.2fr 1.2fr 0.9fr gap-5> ← mb-6 (24px)
│  │       │   ├─ <AttentionCard />
│  │       │   ├─ <WorkingOnYourBehalfCard />
│  │       │   └─ <TeamCard />
│  │       └─ <ActivityFeed />                    ← full-width card
└─ </body>
```

### Page-level layout decisions

- **D-01: Page does NOT add its own max-width or padding.** ClientShell already provides `max-w-[1200px] mx-auto pt-9 px-10 pb-20`. The page renders content directly. (Source: prototype `ClientShell` `4c54d558.js:121-129`; existing `components/shell/ClientShell.tsx`.)
- **D-02: Page outer wrapper is `<div className="relative">`** so the absolute-positioned `<KanjiWatermark>` clips against it. (Source: prototype `ClientDashboard` `05bee446.js:14`.)
- **D-03: No `space-y-*` band stack at the page level.** Section gaps are explicit on each section: GreetingBlock has `mb-7`, ApplicationCard has `mb-6`, the 3-column grid has `mb-6`. The ActivityFeed is the last child with no margin. (Source: prototype `marginBottom: 28` on greeting and `marginBottom: 24` elsewhere — see Spacing Chrome Metric Exceptions.)
